titles = keywords_df["titulo"]
indices = pd.Series(keywords_df.index, index=keywords_df["titulo"])

def keyword_recommendations(title):
    # Obtener el ndice correspondiente al ttulo de la pelcula
    idx = indices.loc[title]
    
    # Obtener los puntajes de similitud de gnero de todas las pelculas
    sim_scores = list(enumerate(cosine_similarity_keywords[idx]))
    
    # Ordenar los puntajes de similitud en orden descendente
    sim_scores = sorted(sim_scores, key=lambda x: x[1], reverse=True)
    
    # Obtener los ndices de las pelculas ms similares (excluyendo la pelcula de consulta)
    sim_scores = sim_scores[1:21]  # Excluyendo el ndice 0, que es la propia pelcula
    
    # Obtener los ndices de las pelculas recomendadas
    movie_indices = [i[0] for i in sim_scores]
    
    # Devolver los ttulos de las pelculas recomendadas
    return titles.iloc[movie_indices]

Ejercicio 2: Recomendador basado en filtrado colaborativo

Sistema de recomendaciones basado en filtrado colaborativo usando los datos de last.fm

Los datos los descargu del link que vimos en clase: http://ocelma.net/MusicRecommendationDataset/lastfm-360K.html

1. Introduccin

En este proyecto, se desarrollar un sistema de recomendaciones basado en filtrado colaborativo utilizando datos de Last.fm. El filtrado colaborativo es una tcnica comnmente utilizada en sistemas de recomendacin, la cual se basa en la agregacin de preferencias o comportamientos de mltiples usuarios para hacer recomendaciones a un usuario especfico. Este enfoque es especialmente til en la recomendacin de contenido donde la interaccin del usuario con los tems (como msica, pelculas, productos) puede ser capturada y analizada.

El objetivo principal es implementar un modelo de filtrado colaborativo, evaluarlo mediante un regresor, proporcionar ejemplos de recomendaciones para usuarios especficos y calcular e interpretar el error cuadrtico medio (MSE) de las predicciones.

6. Creacin del modelo de filtrado colaborativo

Utilizamos el algoritmo Alternating Least Squares (ALS) para el filtrado colaborativo, una tcnica comnmente utilizada en sistemas de recomendacin.

Explicacin terica del ALS

ALS es un algoritmo que se utiliza para completar matrices dispersas, lo que es ideal para sistemas de recomendacin. Funciona alternando entre fijar los factores de usuario y actualizar los factores de tem para minimizar el error cuadrtico medio.

In[71]:
als = ALS(maxIter=10, 
          regParam=0.1, 
          userCol="user_index", 
          itemCol="artist_index", 
          ratingCol="normalized_plays", 
          coldStartStrategy="drop")

# Entrenar el modelo
try:
    model = als.fit(train_data)
    print("Modelo entrenado exitosamente.")
except Exception as e:
    print(f"Error al entrenar el modelo: {e}")

Modelo entrenado exitosamente.

7. Evaluacin del modelo

Evaluamos el rendimiento del modelo utilizando Root Mean Square Error (RMSE).

La mtrica RMSE es til para medir la diferencia entre los valores predichos por el modelo y los valores reales en la escala de la variable dependiente, en este caso, el nmero de reproducciones normalizadas.

Mejoras: Trabajo futuro

La salida de los artistas recomendados para el usuario seleccionado mostro una gran desnormalizacion de los nombres de los artistas. Esto suele pasar mucho el proyectos de BigData (me pas igual en la practica de Spark) all lo suyo es generar un preprocesamiento ms robusto para tratar de pillar y normalizar la mayor cantidad de datos posible, de modo que el string indexer codifique sin duplicados, y as las predicciones puedan ser ms finas.

la primera mejora que propongo es de preprocesamiento y no de afinacin directa de hiperparmetros por el principio de todo data sicentist: Garbage in, garbage out.

Ahora, como segunda opcin de mejora propongo explorar todas las estrategias que conocemos: Validacin cruzada y optimizacin de hiperparmetros, con Grid Search o Random Search.

Tambin sera muy interesante poder incluir caractersticas adicionales de los artistas (gneros, aos de actividad, etc.) porque de acuerdo a lo que estuve leyendo, esto mejora enormemente las recomendaciones y la personalizacin.

Calificaciones directas o, para el caso implicito, clics, tiempo de escucha, podran proporcionar muuucha info sobre cada usuario y esto sin duda ayudaria a generar una personalizacin brutal. Pero claro, esto ya es una mejora que debe estar soportada por alguna plataforma que guarde esta info.. hasta donde supe, Netflix hace esto de guardar no solo el historial de reproduccin sino tambien las pausas, avances y retrocesos.
